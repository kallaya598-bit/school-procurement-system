# ระบบเอกสารพัสดุโรงเรียน - ฉบับอัพเดท

## 🎉 ยินดีต้อนรับ

นี่คือระบบจัดการเอกสารขอซื้อ/ขอจ้างของโรงเรียนแบบอัตโนมัติ ที่อัพเดทแล้ว **พร้อมตราครุฑและตารางสรุปข้อมูล**

---

## ✨ คุณสมบัติใหม่ (อัพเดท 2 มิถุนายน 2569)

### 🎖️ ตราครุฑแสดงบนเอกสาร
- ตราครุฑ (ตราราชการ) แสดงที่มุมซ้ายบนของหน้าบันทึกข้อความ
- ขนาด: 2.25 เซนติเมตร
- แสดงบนหนาแรก และทุกหน้าบันทึกข้อความ

### 📊 ตารางสรุปข้อมูลโครงการ
หลังส่วน "บันทึกข้อความ" จะมีตารางแสดง:
- **ข้อความ**: "ดังเอกสารที่แนบมา"
- **จำนวนโครงการ**: 1
- **ชื่อโครงการและจำนวนเงิน**: อัตโนมัติจากข้อมูลที่กรอก

## 📋 ความต้องการของระบบ

- **Python**: 3.12 ขึ้นไป
- **ไลบรารี**: `python-docx>=1.1.2`
- **ระบบปฏิบัติการ**: Windows, macOS, หรือ Linux
- **พื้นที่ดิสก์**: ประมาณ 50 MB

## 🚀 วิธีติดตั้งและใช้งาน

### 1. ติดตั้ง Python (หากยังไม่มี)
- ไปที่ https://www.python.org/downloads/
- เลือก Python 3.12 หรือสูงกว่า
- ติดตั้งโดยเลือก "Add Python to PATH"

### 2. ติดตั้ง Dependencies
```bash
pip install -r requirements.txt
```

### 3. เรียกใช้ระบบ
```bash
python server.py
```

**ผลลัพธ์**:
```
ระบบพัสดุพร้อมใช้งาน: http://127.0.0.1:8087
```

### 4. เปิดเว็บบราวเซอร์
```
http://127.0.0.1:8087
```

## 🎯 ขั้นตอนการใช้งาน

### ขั้นตอน 1: กรอกข้อมูล
1. กรอกวันที่, เดือน, พ.ศ.
2. เลือกประเภท (ขอซื้อ/ขอจ้าง)
3. กรอกชื่อผู้รับผิดชอบ และตำแหน่ง

5. เพิ่มรายการพัสดุ
6. กรอกชื่อเจ้าหน้าที่ (หากต้องการ)

1. คลิก "สร้างไฟล์ Word"
2. รอสักครู่ให้ระบบประมวลผล
3. คลิก "ดาวน์โหลดไฟล์ Word"

### ขั้นตอน 3: ตรวจสอบและพิมพ์
1. เปิดไฟล์ที่ดาวน์โหลด
2. ตรวจสอบความถูกต้องของข้อมูล
3. ลงนามกำกับ และพิมพ์

## 🌐 ใช้งานบนอินเทอร์เน็ต (Render.com)

### ขั้นตอนที่ 1: อัพโหลดไปยัง GitHub

```bash
# ในโฟลเดอร์ของระบบ
git init
git add .
git commit -m "School procurement system"
git branch -M main
git push -u origin main
```

### ขั้นตอนที่ 2: เชื่อมต่อกับ Render

1. ไปที่ https://render.com
2. เข้าสู่ระบบ
3. คลิก "New Web Service"
4. เลือก GitHub repository
5. ตั้งค่า:
   - **Start Command**: `python server.py`
   - **Build Command**: `pip install -r requirements.txt`
   - **Environment Variable**: `HOST = 0.0.0.0`
6. คลิก "Create Web Service"

### ขั้นตอนที่ 3: เปิดใช้งาน

- ⏳ รอให้ Render สร้างเว็บไซต์ (10-15 นาที)
- ✅ จะได้ URL เช่น: `https://your-app.onrender.com`
- 🌐 ครูทุกคนสามารถเข้าใช้งานจากไปไหนก็ได้

## 🔧 การแก้ไขปัญหาทั่วไป

### ❌ ข้อผิดพลาด: ModuleNotFoundError: No module named 'docx'

**วิธีแก้**:
```bash
pip install python-docx
```

### ❌ ข้อผิดพลาด: Address already in use (พอร์ต 8087)

**วิธีแก้** - ใช้พอร์ตอื่น:
```bash
python server.py --port 8088
```

### ❌ ตราครุฑไม่แสดงในเอกสาร

**วิธีแก้**:
1. ตรวจสอบว่า `images.jpg` อยู่ในโฟลเดอร์เดียวกับ `server.py`
2. ตรวจสอบว่าไฟล์ไม่เสียหาย

### ❌ ตารางสรุปไม่แสดง

**วิธีแก้**:
1. ตรวจสอบว่าได้กรอกชื่อโครงการ
2. ตรวจสอบว่าเพิ่มรายการพัสดุอย่างน้อย 1 รายการ
