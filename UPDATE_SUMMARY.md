# การอัพเดทระบบเอกสารพัสดุโรงเรียน - สรุปการแก้ไข

## 🎯 การเปลี่ยนแปลงที่ทำการอัพเดท

### 1. ✅ เพิ่มตราครุฑบนหน้าบันทึกข้อความ
- **ตำแหน่ง**: มุมซ้ายบนของหน้าบันทึกข้อความ
- **รูปภาพ**: ใช้ไฟล์ `images.jpg` (ตราครุฑ) ที่อัพโหลดมา
- **ขนาด**: ความสูง 2.25 เซนติเมตร
- **การปรับปรุง**: ไฟล์ server.py ตอนนี้ค้นหารูปภาพจากไดเรกทอรีหลัก (root) แทนที่จะค้นหาจากโฟลเดอร์ templates/

### 2. ✅ เพิ่มตารางสรุปข้อมูลโครงการหลังบันทึกข้อความ
- **ตำแหน่ง**: อยู่หลังส่วน "บันทึกข้อความ" บนหน้าแรก
- **เนื้อหาของตาราง**:
  - ข้อความ "ดังเอกสารที่แนบมา" (เป็นหัวข้อก่อนตาราง)
  - จำนวนโครงการ: **1**
  - รายละเอียดโครงการ: ชื่อโครงการและจำนวนเงินทั้งสิ้น
- **รูปแบบ**: ตารางที่สร้างอย่างเป็นทางการ พร้อมสีพื้นหลังสำหรับหัวข้อ

## 📝 รายละเอียดการแก้ไขใน server.py

### การเปลี่ยนแปลง 1: อัพเดท GARUDA_PATH
```python
# เดิม:
GARUDA_PATH = BASE_DIR / "templates" / "image2.png"

# ใหม่:
GARUDA_PATH = BASE_DIR / "images.jpg"
```
ระบบจะค้นหารูปภาพตราครุฑจากไดเรกทอรีหลัก (root directory) ที่ชื่อว่า `images.jpg`

### การเปลี่ยนแปลง 2: เพิ่มฟังก์ชัน add_summary_table()
```python
def add_summary_table(doc, project_name, total_amount, font_size=14):
    """Add a summary table showing project summary with 'ดังเอกสารที่แนบมา' header"""
```

ฟังก์ชันนี้:
- แสดง "ดังเอกสารที่แนบมา" เป็นข้อความหัวข้อ
- สร้างตารางแสดงจำนวนโครงการ (1 โครงการ)
- แสดงรายละเอียดโครงการและจำนวนเงินทั้งสิ้น

### การเปลี่ยนแปลง 3: เรียกใช้ add_summary_table() ในฟังก์ชัน build_docx()
```python
# ใน build_docx() หลังจาก add_memo_header():
add_memo_header(doc, "ขอซื้อพัสดุ/ขอจ้างทำของ/ขอจ้างเหมาบริการ", "ศธ04299.37/จัดซื้อจัดจ้าง", data, body_size=compact_size, title_size=22, garuda_position="left")

# เพิ่มบรรทัดใหม่:
add_summary_table(doc, data["project"], total, font_size=14)
```

## 📦 ไฟล์ที่จำเป็น

- ✅ **server.py** - ไฟล์หลักที่อัพเดท (พร้อมฟังก์ชันใหม่)
- ✅ **images.jpg** - รูปตราครุฑ (ต้องอยู่ในไดเรกทอรีหลัก)
- ✅ **index.html** - ไฟล์ HTML
- ✅ **app.js** - ไฟล์ JavaScript
- ✅ **styles.css** - ไฟล์ CSS
- ✅ **requirements.txt** - ไฟล์ dependencies
- ✅ **runtime.txt** - เวอร์ชัน Python
- ✅ **Procfile** - สำหรับ Render/Heroku
- ✅ **render.yaml** - การตั้งค่า Render

## 🚀 วิธีใช้งาน

### รันในเครื่องคอมพิวเตอร์
```bash
pip install -r requirements.txt
python server.py
```

จากนั้นเปิด: `http://127.0.0.1:8087`

### อัพโหลดไปใช้บนเว็บ (Render.com)
1. อัพโหลดไฟล์ทั้งหมดไปยัง GitHub
2. เชื่อมต่อ Render กับ GitHub
3. Render จะอ่าน `render.yaml` และตั้งค่าอัตโนมัติ

## ⚠️ หมายเหตุสำคัญ

- **ตำแหน่งของ images.jpg**: ไฟล์ `images.jpg` ต้องอยู่ในไดเรกทอรีเดียวกับ `server.py` (root directory)
- **ตราครุฑจะแสดงบนทุกหน้าบันทึกข้อความ**: ทั้งหน้าแรก หน้าแต่งตั้งคณะกรรมการ และหน้าขอความเห็นชอบ
- **สรุปข้อมูลจะแสดงเพียงหน้าแรก**: ในหน้าหลัก (บันทึกข้อความแรก) หลังจากส่วน "บันทึกข้อความ"
